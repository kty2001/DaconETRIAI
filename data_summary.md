# 라이프로그 데이터를 활용한 수면 및 심리 지표 예측 — 데이터 정리

## 1. 대회 개요

700일 분량의 시계열 라이프로그 데이터와 450일분의 레이블을 이용해 **7가지 수면/심리 지표(Q1~Q3, S1~S4)** 를 이진 분류(0/1)로 예측하는 과제.

### 예측 대상 지표

| 지표 | 분류 | 설명 | 1의 의미 |
|------|------|------|----------|
| Q1 | 설문 | 수면의 질 (기상 직후) | 평소보다 좋음 |
| Q2 | 설문 | 피로도 (취침 직전) | 평소보다 피로가 적음 |
| Q3 | 설문 | 스트레스 (취침 직전) | 평소보다 스트레스 낮음 |
| S1 | 센서 | 총 수면 시간 (TST) | 권장 기준 충족 |
| S2 | 센서 | 수면 효율 (SE) | 권장 기준 충족 |
| S3 | 센서 | 수면 지연 시간 (SOL) | 권장 기준 충족 |
| S4 | 센서 | 수면 중 각성 시간 (WASO) | 권장 기준 충족 |

## 5. 라이프로그 parquet 데이터

모든 파일의 공통 구조: `subject_id + timestamp + 측정값`  
전체 수집 기간: **2024-06-03 ~ 2024-11-19**

### 스마트폰 센서 (`m_` 접두사)

| 파일 | 행 수 | 컬럼 | 값 형태 | 설명 |
|------|-------|------|---------|------|
| mACStatus | 939,896 | `m_charging` | 0/1 (int) | 충전 중 여부 |
| mActivity | 961,062 | `m_activity` | 코드 (int) | 활동 유형 (3=정지, 4=걷기, 7=달리기) |
| mAmbience | 476,577 | `m_ambience` | 리스트 | 주변 소리 분류명 + 확률값 |
| mBle | 21,830 | `m_ble` | 리스트 | 주변 BLE 기기 주소 + RSSI |
| mGps | 800,611 | `m_gps` | 리스트 | GPS 좌표/속도/고도 (익명화) |
| mLight | 96,258 | `m_light` | float (lux) | 조도 (0 ~ 334,306) |
| mScreenStatus | 939,653 | `m_screen_use` | 0/1 (int) | 화면 켜짐 여부 |
| mUsageStats | 45,197 | `m_usage_stats` | 리스트 | 앱 이름 + 사용 시간(초) |
| mWifi | 76,336 | `m_wifi` | 리스트 | 주변 WiFi AP BSSID + RSSI |

### 스마트워치 센서 (`w_` 접두사)

| 파일 | 행 수 | 컬럼 | 값 형태 | 설명 |
|------|-------|------|---------|------|
| wHr | 382,918 | `heart_rate` | 리스트 (int) | 분당 측정된 심박수 배열 |
| wLight | 633,741 | `w_light` | float | 워치 조도 센서 |
| wPedo | 748,100 | `step`, `speed`, `distance`, `burned_calories` 등 | float | 걸음수/속도/거리/칼로리 |

### 중첩 구조 컬럼 주의사항

`mAmbience`, `mBle`, `mGps`, `mWifi`, `mUsageStats`, `wHr` 컬럼은 **리스트/딕셔너리 중첩 구조**로 저장되어 있어 집계 전 파싱이 필요.

```python
# wHr 예시: 1분 타임스탬프당 초 단위 심박수 배열
timestamp=2024-06-26 12:23:00 → heart_rate=[134, 134, 135, 133, ...]

# mAmbience 예시: 소리 분류 + 확률
timestamp=2024-06-26 13:00:10 → [['Music', 0.309], ['Vehicle', 0.082], ...]

# mUsageStats 예시: 앱별 사용 시간(초)
→ [{'app_name': 'NAVER', 'total_time': 549}, ...]
```

## 7. 프로토타입 계획

### 사용 데이터
- CSV만 사용 (parquet 미사용) → 피처가 제한적이나 빠른 검증 가능

### 추출 가능 피처

| 피처 | 설명 |
|------|------|
| subject 인코딩 | 개인 식별자 (label encoding) |
| subject별 target 평균 | 각 인물의 역대 Q1~S4 평균 (개인 성향 반영) |
| 요일 | sleep_date 기준 (주말/평일 패턴) |
| 월 | 계절적 패턴 |
| 주차 | 주 단위 패턴 |

### 모델

| 모델 | 역할 |
|------|------|
| Subject 평균 기반 분류 | 베이스라인 (개인 성향만 반영) |
| **LightGBM** | 메인 모델 (날짜 피처 + subject 인코딩) |
| logistic regression | 피처 중요도 확인 |

- 타깃 7개를 **개별 이진 분류기**로 각각 학습
- StratifiedKFold CV로 과적합 여부 확인

### 코드 구조

```
feature_engineering.py   # 피처 생성
train_predict.py          # 모델 학습 및 예측
submission.py             # 결과 저장
```
